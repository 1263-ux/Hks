# 《山西古建保卫战》美术全面升级计划

## 当前状态

| 元素 | 现状 | 尺寸 | 问题 |
|------|------|------|------|
| 玩家 | 静态精灵图 | 32×32 | 无动画、太小 |
| 怪物 | 静态精灵图×5 | 16-28px | 无动画、辨识度不够 |
| 古建 | 静态精灵图 | 100×70 | 无受损状态变化 |
| 背景 | 480×270 平铺 | 单层 | 缺少层次感 |
| 技能 | 纯几何图形 | - | 冲击波/震波太简陋 |
| UI | Phaser 矩形+文字 | - | 血条/卡片/按钮风格单调 |
| 粒子 | 纯色小圆 | - | 颜色单一 |

## 升级原则

- **全代码生成**：Canvas2D 绘制，零外部文件
- **体积预算**：目标 JS 包 < 2MB（当前 1.52MB，剩余约 480KB）
- **像素风格**：16×16 ~ 48×48 为主，配合粗像素感
- **古建国风**：棕色/金色/红色主色调，呼应山西古建主题

## 阶段 1：角色动画（预计 +5KB）

### 1.1 玩家行走动画（4 帧，32×32）

```
帧0: 右脚在前，身体略倾
帧1: 中立站姿
帧2: 左脚在前，身体略倾
帧3: 中立站姿

14fps 循环播放，移动时播放，静止时停在第1帧
```

### 1.2 怪物动画（各 2-4 帧）

| 怪物 | 动画 | 帧数 |
|------|------|------|
| 白蚁怪 | 身体蠕动（上下挤压） | 2 |
| 风蚀怪 | 旋转（角度递增） | 4 |
| 酸雨怪 | 气泡上浮 + 身体波动 | 3 |
| 火焰怪 | 火焰摇曳（内外焰缩放） | 3 |
| 冻融怪 | 冰晶闪光（高光位移） | 2 |

### 1.3 怪物死亡动画

- 白蚁怪：挤压变扁 → 爆散
- 风蚀怪：旋转加速 → 消散
- 酸雨怪：气泡破裂 → 溅开
- 火焰怪：先膨胀 → 熄灭
- 冻融怪：裂纹扩大 → 碎裂

### 1.4 古建受损状态（4 个阶段）

```
100% → 完整古建（当前）
75%  → 屋顶略微破损，一处木柱倾斜
50%  → 屋顶缺角，木柱断裂一根
25%  → 外墙裂纹，屋顶大面积破损
0%   → 坍塌废墟
```

按最低血条的百分比切换显示。

## 阶段 2：技能特效重做（预计 +3KB）

### 2.1 木构加固 — 木梁冲击波

```
当前: 棕色矩形
升级: 木梁纹理（木纹线条），前端尖角，后方碎木粒子拖尾
      Lv.2 梁上出现金色加固纹
      Lv.3 多处榫卯节点金色闪光
```

### 2.2 石材修补 — 石粉震波

```
当前: 灰色扩散圈
升级: 多层石粉碎屑向外爆开 + 地面裂纹纹理
      Lv.2 石粉量增加
      Lv.3 碎石块伴随震波飞出
```

### 2.3 防水封护 — 水纹护罩

```
当前: 蓝色扩散圈
升级: 多层同心水纹波 + 水珠粒子
      Lv.2 水纹波数量×2
      Lv.3 护罩闭合为半透明蓝色穹顶（椭圆弧线）
```

### 2.4 防虫处理 — 药雾区域

```
当前: 半透明绿圆
升级: 不规则雾团粒子 + 飘散效果
      Lv.2 雾团更浓密
      Lv.3 雾团外围有驱虫草药碎屑飘落
```

### 2.5 彩绘修复 — 颜料弹

```
当前: 粉色小圆
升级: 拖尾彩虹粒子（红橙黄绿青蓝紫渐变）
      命中目标时颜料溅射
      Lv.2 爆炸时彩色颜料四溅
      Lv.3 弹射轨迹带金色闪光
```

### 2.6 自动普攻

```
当前: 蓝色辉光点
升级: 细长箭矢形 + 白/蓝渐变拖尾
```

## 阶段 4：粒子与环境（预计 +2KB）

### 4.1 通用粒子系统

```
- 受击粒子: 白色碎屑
- 死亡粒子: 怪物颜色碎片（已实现，增强为更多粒子+颜色渐变）

- 经验拾取: 小光点吸入玩家
```

### 4.2 环境层

```
- 地图四角增加小型古建装饰物（石灯/经幢/石碑）
- 古建周围增加一圈石板地（已部分实现）
- 微风吹过时草地震荡波动（不可见但粒子暗示）
```

### 4.3 天气/氛围

```
- 最后 1 分钟画面边缘出现暗角（紧张感）
- 古建受损严重时天空变暗红色调
```

## 阶段 5：性能与适配（预计 +1KB）

### 5.1 纹理图集

```
将所有小纹理合并到一张 Canvas 上（spritesheet），减少 Phaser 纹理数量
```

### 5.2 缩放适配

```
所有精灵为 2x 像素绘制（如 64×64 实际绘制 128×128 再缩回），
在高 DPI 屏幕上更清晰
```

## 体积预估

| 阶段 | 新增代码 | 新增纹理 |
|------|---------|---------|
| 角色动画 | ~3KB | ~2KB（多帧） |
| 技能特效 | ~2KB | ~1KB（粒子纹理） |
| UI 美术化 | ~1.5KB | ~0.5KB（边框纹理） |
| 粒子环境 | ~1KB | ~1KB |
| 总计 | ~7.5KB | ~4.5KB |

**预计最终 JS 包 < 1.53MB**，远低于 8MB 限制。

## 建议优先做

**阶段 1+2 同步开工**：古建受损状态 + 技能特效重做。这两个对玩家视觉冲击最大——古建被攻击时外观变化直观反映危险程度，技能特效是玩家割草爽感的主要来源。
